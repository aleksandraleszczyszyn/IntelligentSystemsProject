Detailed Description

Represents an object category, e.g., apple, bowl, bell-pepper, etc.

Open issues:

  • a category should hold/handle its' own histogram. In this version the category' histogram is computed every time a new view-category prediction is made. A better approach could hold and maintain the histogram instead of computing it every time.
